# Archz Documentation

This folder documents the repository as it exists today, while preserving the intended architecture called out in the root `README.md`.

## Start Here

- [Architecture](ARCHITECTURE.md): solution structure, services, building blocks, and current implementation status.
- [Service Patterns](SERVICE-PATTERNS.md): how controllers, commands, queries, validators, startup files, and domain objects are organized.
- [Data Access](DATA-ACCESS.md): EF Core, migrations, DbContexts, repositories, Identity storage, and pending persistence work.
- [Local Development](LOCAL-DEVELOPMENT.md): tools, Docker Compose, ports, build commands, and EF CLI notes.
- [Quality](QUALITY.md): validation commands, current build state, test status, and expected quality gates.
- [Decisions](DECISIONS.md): accepted architecture decisions and unresolved follow-up decisions.

## Documentation Principle

Document current implementation first. When the repository shows an intended pattern that is not complete yet, mark it as planned, pending, or partially implemented.
